Seatwise v2 - Project Directories Explanation
=============================================

Here is a breakdown of what each directory in the Seatwise project is used for:

1. .agent
   Contains intelligence files, configuration, skills, and behavior rules for the AI agents operating within this project.

2. .claude
   Configuration directory for the Claude AI assistant integration.

3. .git 
   The Git version control repository that stores the project's commit history and source control data.

4. .next
   Automatically generated by Next.js. This folder contains the build output, cache, and compiled files needed to run the application.

5. .vscode
   Configuration settings, suggested extensions, and workspace rules for the Visual Studio Code editor.

6. app
   The core of the Next.js application using the App Router. It contains the structure for pages, layouts, and backend API routes/server actions.

7. components
   Stores reusable React frontend components (such as buttons, dialogs, or 3D venue renders) built with Tailwind CSS, Radix UI, etc.

8. hooks
   Custom React hooks used to extract and share repeatable frontend logic, data fetching, or state management across multiple components.

9. lib
   Core utility libraries, application configurations, and instantiations for third-party services (e.g., the Prisma database client or Redis/Upstash connection setups).

10. node_modules
    Auto-generated directory containing all the downloaded npm dependencies and packages required by the project.

11. prisma
    Contains the Prisma ORM schema definitions (schema.prisma) and database migration files used to interact with the Neon Serverless PostgreSQL database.

12. public
    Static assets that are served directly to the user's browser without any processing, such as images, fonts, icons, and 3D GLTF models (e.g., amphitheater models).

13. scripts
    Custom standalone automation, data-processing, or maintenance scripts (such as queue cleanup or system deployment utilities).

14. test
    Contains automated test files, test fixtures, and testing configurations used to ensure the reliability and correctness of the codebase.

15. utils
    General-purpose helper functions, data formatting utilities, and generic scripts that can be used universally across the application.
